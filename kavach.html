Full-Stack
Architecture.

Kavach is built on a high-performance Python backend and a modern React frontend. It achieves over 80% test coverage via Pytest and Jest, ensuring critical generation and billing pathways never fail.

Infrastructure Stack

  • Backend Engine FastAPI (Python 3.11+)
  • Database PostgreSQL 15+ & SQLAlchemy 2.0
  • Frontend App Next.js 14 (App Router)
  • UI Primitives shadcn/ui + Tailwind CSS
  • Authentication NextAuth.js 4 + Argon2
  • Payment Gateway Razorpay SDK 1.4+
  • Export Engines WeasyPrint + python-docx

Frequently Asked.

Everything you need to know about using Kavach for your legal documents.

Yes. All document templates are drafted with reference to Indian contract law, the Indian Contract Act 1872, and relevant state-specific regulations. Documents include jurisdiction clauses, proper party identification, and consideration terms. We recommend review by a legal professional for high-stakes agreements.

Kavach supports stamp duty rates for 14 Indian states: Maharashtra, Karnataka, Tamil Nadu, Uttar Pradesh, Gujarat, Rajasthan, Delhi, West Bengal, Telangana, Andhra Pradesh, Kerala, Madhya Pradesh, Punjab, and Haryana. More states are being added regularly.

Kavach uses Groq's LPU inference running llama-3.3-70b-versatile as the primary engine. If Groq is unavailable, it automatically falls back to NVIDIA NIM (meta/llama-3.3-70b-instruct). All generation happens via API — no data is used for model training.
